Dev Containers & Codespaces

Onboard in minutes, not days

What Are Dev Containers?

Dev Container: a Docker-based dev environment defined by a JSON config in your repo. Language runtimes, tools, extensions, and services so every contributor gets an identical setup.

GitHub Codespaces: hosts Dev Containers in the cloud. Click a button, get a full VS Code environment in your browser in under 2 minutes.

📖 containers.dev · docs.github.com/en/codespaces

The Problem

New contributor joins your Imagine Cup team.

They spend a full day installing Python, Node, PostgreSQL, CLI tools, VS Code extensions, environment variables…

Then it still doesn't work on their machine.

GitHub Actions CI/CD

Push to main, deploy to production

What Is GitHub Actions?

GitHub Actions is a CI/CD platform built into GitHub. Define workflows in YAML that run on every push, pull request, schedule, or manual trigger. Each workflow runs on GitHub-hosted runners (Ubuntu, Windows, macOS) with full Docker support.

  • Free for public repos, generous free tier for private
  • Marketplace: 20,000+ pre-built actions to compose
  • Matrix builds: test across multiple versions/OSes in parallel
  • Secrets management: encrypted env vars, OIDC for cloud auth

📖 docs.github.com/en/actions

Pipeline at a Glance

JobTriggerWhat It Does
changesEvery pushDetect which folders changed (skip unnecessary deploys)
dependency-reviewPRs onlyBlock PRs with vulnerable dependencies
ciEvery push + PRLint → format → type-check → test with coverage
terraformMain onlyPlan + apply infrastructure changes
deployMain onlyBuild Docker → push to ACR → update Container App

CI Job: Lint, Type-Check, Test

.github/workflows/deploy.yml

  ci:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16-alpine
        env:
          POSTGRES_USER: postgres
          POSTGRES_PASSWORD: postgres
          POSTGRES_DB: learntocloud
        ports:
          - 5432:5432
    env:
      DATABASE_URL: postgresql+asyncpg://postgres:postgres@localhost:5432/learntocloud
    defaults:
      run:
        working-directory: api
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version-file: "api/pyproject.toml"

      - name: Install uv
        uses: astral-sh/setup-uv@v7

      - name: Install dependencies
        run: uv sync --locked

      - name: Ruff lint
        uses: astral-sh/ruff-action@v3
        with:
          src: "./api"
          args: "check"

      - name: Ruff format check
        uses: astral-sh/ruff-action@v3
        with:
          src: "./api"
          args: "format --check"

      - name: Type check (ty)
        run: uv run ty check --python .venv/bin/python

      - name: Run tests with coverage
        run: uv run pytest tests/ -v --tb=short --cov=. --cov-report=xml

Smart Deploys

Path filtering: only deploy when relevant files change

  changes:
    runs-on: ubuntu-latest
    steps:
      - uses: dorny/paths-filter@v3
        id: filter
        with:
          filters: |
            deploy:
              - 'api/**'
              - 'content/**'
              - 'infra/**'

Concurrency control: new push cancels in-progress deploys

concurrency:
  group: deploy-${{ github.ref }}
  cancel-in-progress: true

GitHub Advanced Security

Ship fast without shipping vulnerabilities

What Is GitHub Advanced Security?

GHAS is a suite of security features integrated directly into your GitHub workflow. No external tools, no context switching.

FeatureWhat It DoesDocs
CodeQL Semantic static analysis that treats code as data and queries it for vulnerability patterns (SQL injection, XSS, auth bypass, etc.) Code scanning
Dependency Review Diff-aware check on PRs that flags new dependencies with known CVEs before they merge Dependency review
Dependabot Automatically opens PRs to update vulnerable or outdated dependencies Dependabot
Security Advisories Private vulnerability reporting through GitHub, not public issues Security advisories

Free for public repos. Included with GitHub Enterprise Cloud for private repos.

Four Layers of Automated Security

🔍 CodeQL

Static analysis on every push + PR + weekly schedule. Catches OWASP Top 10 patterns.

📦 Dependency Review

Blocks PRs introducing dependencies with moderate+ vulnerabilities.

🤖 Dependabot

Weekly automated PRs across 4 ecosystems: Python, npm, Actions, Terraform.

🔐 Security Policy

SECURITY.md → private vulnerability disclosure via GitHub Security Advisories.

None of this required building anything. 3 YAML files + 1 markdown doc.

CodeQL: Static Analysis

.github/workflows/codeql.yml

name: CodeQL

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
  schedule:
    # Weekly scan - catches newly disclosed CVE patterns
    - cron: "0 6 * * 1"

permissions:
  security-events: write
  contents: read

jobs:
  analyze:
    name: Analyze (${{ matrix.language }})
    runs-on: ubuntu-latest
    strategy:
      fail-fast: false
      matrix:
        language: [python, javascript]
    steps:
      - uses: actions/checkout@v4

      - name: Initialize CodeQL
        uses: github/codeql-action/init@v4
        with:
          languages: ${{ matrix.language }}
          queries: security-extended

      - name: Autobuild
        uses: github/codeql-action/autobuild@v4

      - name: Perform CodeQL Analysis
        uses: github/codeql-action/analyze@v4
        with:
          category: "/language:${{ matrix.language }}"

Dependabot: Auto-Update Everything

.github/dependabot.yml

version: 2
updates:
  # Python dependencies (uv ecosystem)
  - package-ecosystem: "uv"
    directory: "/api"
    schedule:
      interval: "weekly"
    commit-message:
      prefix: "chore(deps):"
    labels:
      - "dependencies"
      - "python"

  # npm dependencies (Tailwind CSS)
  - package-ecosystem: "npm"
    directory: "/api"
    schedule:
      interval: "weekly"

  # GitHub Actions - keep actions pinned to latest
  - package-ecosystem: "github-actions"
    directory: "/"
    schedule:
      interval: "weekly"

  # Terraform providers
  - package-ecosystem: "terraform"
    directory: "/infra"
    schedule:
      interval: "weekly"

Dependency Review: Gate on PRs

.github/workflows/deploy.yml (dependency-review job)

  dependency-review:
    if: github.event_name == 'pull_request'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/dependency-review-action@v4
        with:
          fail-on-severity: moderate

✓ Runs on every PR, catches vulns before they merge

✓ fail-on-severity: moderate blocks moderate, high, and critical

7 lines of YAML. That's it.

Security Policy

SECURITY.md

# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT open a public GitHub issue.**
2. Use GitHub Security Advisories to privately report
   the vulnerability.
3. Include steps to reproduce, affected components,
   and potential impact.

## Response Timeline

- **Acknowledgment**: Within 3 business days.
- **Assessment**: Severity and impact within 7 business days.
- **Fix**: Critical/high severity prioritized for next release.

GitHub automatically shows the "Security" tab and links to this file. Professional signal to judges and users.

Copilot Customization

Make AI know your project

What Is Copilot Customization?

GitHub Copilot can be customized with project-specific knowledge via markdown files checked into your repo. These files teach Copilot your conventions, workflows, and domain expertise.

ConceptWhat It IsDocs
AGENTS.md Project guidelines: tech stack, architecture, code style, build commands, conventions Customization
Custom Instructions Persistent context about your stack, coding style, and conventions Custom instructions
Skills (SKILL.md) Reusable multi-step procedures triggered by natural language phrases VS Code customization
Prompt Files (.prompt.md) Structured reusable prompts, like saved expert workflows Prompt files
Agents (.agent.md) Autonomous task runners with restricted tool access Agent mode

The Problem with Generic AI

Copilot is powerful, but out of the box it doesn't know:

  • Your deploy process
  • Your database schema
  • Your team's conventions
  • How to check if prod is healthy
  • How to ship a change end-to-end

You can teach it all of this with files in your repo.

Four Layers of Customization

LayerLocationPurpose
AGENTS.md AGENTS.md Project guidelines Copilot reads on every interaction
Skills .github/skills/ Reusable procedures triggered by natural language
Prompt Files .github/prompts/ Structured multi-step workflows
Agents .github/agents/ Autonomous multi-step task runners

All stored as markdown files in your repo. Version controlled. Shared with the team.

Copilot Workflows

AI agents running on a schedule in GitHub Actions

What Are Copilot Workflows?

Copilot Workflows (powered by gh-aw) let you write GitHub Actions workflows in natural language markdown instead of YAML. A Copilot agent executes the workflow using tools you grant it: GitHub API, bash commands, web fetch, MCP servers.

  • Markdown in, YAML out: compiles to deterministic .lock.yml files
  • Scheduled or manual: cron triggers, workflow_dispatch, or event-driven
  • Safe outputs: auto-closes old issues, applies labels, title prefixes
  • Tool restrictions: whitelist exactly which bash commands or APIs the agent can use

📖 githubnext.com/projects/copilot-workflows

Workflows in Markdown, Not YAML

.github/workflows/content-link-auditor.md

---
on:
  schedule: weekly on monday
  workflow_dispatch:

description: >
  Weekly audit of all external URLs in Learn to Cloud
  content YAML files. Checks for broken links and redirects.

engine: copilot

tools:
  bash: ["cat", "find", "grep", "head", "tail",
         "wc", "sort", "curl", "python3"]
---

Write the goal in natural language. The AI figures out the how.

Compiled to .lock.yml by gh aw compile for deterministic execution.
