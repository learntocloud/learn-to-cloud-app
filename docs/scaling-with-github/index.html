# Project Instructions

## Tech Stack
- Backend: FastAPI 0.139+ + Python 3.13, async throughout
- Database: PostgreSQL 16 via async SQLAlchemy 2.0
- Frontend: Jinja2 templates + HTMX + Tailwind CSS v4
- Auth: GitHub OAuth via Authlib
- Infra: Azure Container Apps, Terraform

## Architecture
Routes (HTTP) → Services (Business Logic) → Repositories (DB)

## Code Style
- Ruff for linting + formatting (line-length 88)
- ty for type checking
- Async/await everywhere

GitHub Actions: CI Pipeline

JobTriggerWhat It Does
changesPush + PRDetect code and infrastructure changes
dependency-reviewPRs onlyBlock PRs with vulnerable dependencies
ciCode changesStatic checks, content validation, migrations, and tests
terraform-ciInfra changesFormat, initialize, validate, and test Terraform
terraformMain + manualPlan infrastructure; apply only on main
deployMain + manualBuild images, migrate, and update Azure services

Path Filtering

Application deploys run only for code, package, or infrastructure changes

Concurrency Control

PR runs cancel when superseded; active production deploys finish safely

The CI Job

.github/workflows/deploy.yml

  ci:
    needs: [changes]
    if: needs.changes.outputs.code == 'true'
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16-alpine
        env:
          POSTGRES_USER: postgres
          POSTGRES_PASSWORD: postgres
          POSTGRES_DB: learntocloud
        ports: ["5432:5432"]

    steps:
      - uses: actions/checkout@v7
      - uses: actions/setup-python@v6
      - uses: astral-sh/setup-uv@v8
      - run: uv sync --all-packages --locked
      - run: uv run poe static
      - run: uv run poe package-smoke
      - run: cd api && uv run alembic upgrade head
      - run: uv run poe test

Relevant pushes and pull requests run one workspace-wide quality gate against PostgreSQL.

Skill: "check prod"

# Production Health Check

14 checks. One verdict. All read-only. Uses Azure CLI.

## Verdict Logic
🔴 Critical - Non-200 readiness, 5xx errors, DB CPU > 80%
⚠️  Warning  - P95 > 500ms, DB CPU 50-80%, failed availability
✅ Healthy  - None of the above

## Coverage
- Readiness and Azure resource health
- Availability, request health, and error trends
- Exceptions, dependencies, and database metrics
- Container metrics, stability, alerts, and console errors
- Authentication and verification business signals

Operations as Natural Language

"query db"Production PostgreSQL queries with Entra ID auth

"ship it"Lint → commit → push → monitor deploy → verify prod
